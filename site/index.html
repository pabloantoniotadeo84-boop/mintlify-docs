From API key to verifiable agent in three steps.

No new infrastructure to learn. Register, issue, attest. Every action carries proof.

01

Register your agent

Each agent gets a SPIFFE identity bound to an Ed25519 key pair that only Vane controls.

02

Issue a passport

Vane signs a short-lived CAP+JWT encoding the agent's scopes and full delegation chain.

03

Every action is attested

Each call becomes a signed record in a hash-chained, Merkle-backed log anyone can verify.

This is a Vane passport.

A CAP+JWT credential any server can verify offline, in milliseconds, with nothing but Vane's public key. No callback. No trust in the caller.

Everything an identity layer needs.

Six primitives, one public key. No agents, no daemons watching you. Just proof.

Cryptographic identity

Every agent gets a SPIFFE workload identity bound to an Ed25519 key pair Vane controls.

Offline verification

Verify any passport with one public key. No callback to Vane, no network round-trip.

Delegation chains

Prove exactly who authorized an agent, from organization down to the individual instance.

Instant revocation

Kill a compromised passport immediately through the revocation list and OCSP responder.

Merkle tree attestation

Every action is a signed record. Prove inclusion in O(log n) without the full chain.

Framework integrations

Drop into LangChain, CrewAI, the OpenAI Agents SDK, and any MCP server with a few lines.

Built for the stack you already run.

Vane meets your agents where they live. Wrap a framework callback or guard an MCP server, and every tool call is signed and attested automatically.
